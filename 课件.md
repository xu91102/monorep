# Monorepo 工程管理实战指南

## 一、Monorepo 概述

### 1.1 Multirepo vs Monorepo

![monorepo vs multirepo](https://resource.duyiedu.com/yuanjin/202509190953043.svg)

**Multirepo（多仓库）**：
- 每个项目/包独立仓库
- 独立版本管理
- 依赖通过 npm 发布和安装

**Monorepo（单仓库）**：
- 多个项目/包在同一仓库
- 统一版本管理和依赖
- 代码共享更便捷

### 1.2 常见 Monorepo 管理工具

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| **pnpm** | 磁盘高效、快速、支持 workspace | 中小型项目 |
| **Turborepo** | 智能缓存、并行构建 | 需要高性能构建 |
| **Nx** | 功能全面、可视化工具 | 大型企业项目 |
| **Lerna** | 老牌工具、版本管理 | 传统项目 |
| **Rush** | 微软出品、企业级 | 超大型项目 |
| **Yarn Workspaces** | 简单易用 | 小型项目 |

### 1.3 本项目技术栈

```
pnpm workspaces + Turborepo + TypeScript + Vitest + Changesets
```

## 二、pnpm Workspace 配置

### 2.1 初始化 Workspace

创建 `pnpm-workspace.yaml`：

```yaml
# pnpm-workspace.yaml
packages:
  - "packages/*"  # 公共包目录
  - "apps/*"      # 应用目录
```

### 2.2 项目目录结构

```
monorep/
├── apps/                    # 应用层
│   ├── backend/            # 后端服务 (Express + TypeScript)
│   └── frontend/           # 前端应用 (Vue 3 + Vite)
├── packages/               # 公共包层
│   ├── cli/               # CLI 工具
│   ├── components/        # Vue 组件库
│   ├── types/             # 共享类型定义
│   └── utils/             # 工具函数库
├── scripts/               # 脚本工具
├── docker/                # Docker 配置
├── e2e/                   # E2E 测试
├── .github/workflows/     # CI/CD 配置
├── pnpm-workspace.yaml    # workspace 配置
├── turbo.json             # Turborepo 配置
├── package.json           # 根配置
└── tsconfig.json          # TypeScript 基础配置
```

### 2.3 pnpm 常用命令

#### 工程级命令（在根目录执行）

```shell
# 安装所有依赖
pnpm install

# 在根目录执行命令
pnpm -w <command>
pnpm --workspace-root <command>

# 示例：在根目录安装开发依赖
pnpm -Dw add typescript
```

#### 子包命令

```shell
# 方式 1：进入子目录执行
cd apps/backend
pnpm dev

# 方式 2：使用 -C 参数
pnpm -C apps/backend dev

# 方式 3：使用 --filter（推荐）
pnpm --filter @monorep/backend dev
pnpm --filter @monorep/frontend build

# 为特定包添加依赖
pnpm add lodash --filter @monorep/utils
pnpm add -D vitest --filter @monorep/backend
```

#### 批量执行命令

```shell
# 所有包执行 build
pnpm -r build

# 所有包执行 test
pnpm -r test

# 并行执行（更快）
pnpm -r --parallel dev
```

## 三、环境版本锁定

### 3.1 配置 engines

在 `package.json` 中指定 Node.js 和包管理器版本：

```json
{
  "engines": {
    "node": ">=18.0.0",
    "pnpm": ">=8.0.0"
  },
  "packageManager": "pnpm@8.15.0"
}
```

### 3.2 强制版本检查

创建 `.npmrc` 文件：

```properties
# .npmrc
engine-strict=true
```

**作用**：
- 确保团队使用统一的 Node.js 和 pnpm 版本
- 避免因版本差异导致的兼容性问题
- 安装依赖时自动检查版本

```json
    "noUncheckedSideEffectImports": true,

    "skipLibCheck": true

  "exclude": ["node_modules", "dist"]
```

### 4.3 子包 tsconfig.json

#### 后端配置 (apps/backend/tsconfig.json)

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "types": ["node"],
    "lib": ["esnext"],
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src"]
```

#### 前端配置 (apps/frontend/tsconfig.json)

```json
{

  "compilerOptions": {
    "types": ["node"],
    "lib": ["esnext", "DOM", "DOM.Iterable"]
  },
  "include": ["src"]
}
```

#### 公共包配置 (packages/utils/tsconfig.json)

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src"]
}
```

### 4.4 TypeScript 配置说明

| 配置项 | 说明 |
|--------|------|
| `extends` | 继承根配置，保持一致性 |
| `strict` | 启用所有严格类型检查 |
| `declaration` | 生成 .d.ts 类型声明文件 |
| `sourceMap` | 生成 source map 便于调试 |
| `skipLibCheck` | 跳过库文件检查，提升速度 |
| `isolatedModules` | 确保每个文件可独立编译 |

## 六、Turborepo 构建优化

### 6.1 Turborepo 简介

Turborepo 是一个高性能的 monorepo 构建系统，提供：
- 智能任务缓存
- 并行任务执行
- 依赖关系管理
- 远程缓存支持

### 6.2 安装

```shell
pnpm -Dw add turbo
```

### 6.3 配置文件 (turbo.json)

```json
{
  "$schema": "https://turbo.build/schema.json",
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "test": {
      "dependsOn": ["^build"],
      "outputs": ["coverage/**"]
    },
    "lint": {
      "outputs": []
    },
    "typecheck": {
      "dependsOn": ["^build"],
      "outputs": []
    },
    "clean": {
      "cache": false
    }
  }
}
```

### 6.4 配置说明

| 配置项 | 说明 |
|--------|------|
| `dependsOn: ["^build"]` | 依赖其他包的 build 任务完成 |
| `outputs` | 指定缓存的输出目录 |
| `cache: false` | 禁用缓存（如 dev、clean） |
| `persistent: true` | 长期运行的任务（如 dev server） |

### 6.5 使用命令

```shell
# 构建所有包
pnpm build  # 实际执行 turbo run build

# 运行测试
pnpm test

# 开发模式
pnpm dev

# 清除缓存
pnpm turbo clean
```

### 6.6 Turborepo 优势

1. **增量构建**：只构建变更的包
2. **并行执行**：自动并行执行独立任务
3. **智能缓存**：缓存构建结果，避免重复构建
4. **依赖感知**：自动处理包之间的依赖关系

## 八、子包间依赖管理

### 8.1 Workspace 协议

pnpm 支持使用 `workspace:` 协议引用本地包：

```json
{
  "dependencies": {
    "@monorep/utils": "workspace:*",
    "@monorep/types": "workspace:^1.0.0"
  }
}
```

### 8.2 协议说明

| 协议 | 说明 | 示例 |
|------|------|------|
| `workspace:*` | 使用当前版本 | 开发时推荐 |
| `workspace:^` | 使用兼容版本 | 发布时推荐 |
| `workspace:~` | 使用补丁版本 | 严格版本控制 |

### 8.3 实际应用示例

#### apps/frontend/package.json

```json
{
  "name": "@monorep/frontend",
  "dependencies": {
    "@monorep/components": "workspace:*",
    "@monorep/utils": "workspace:*",
    "vue": "^3.4.0"
  }
}
```

#### apps/backend/package.json

```json
{
  "name": "@monorep/backend",
  "dependencies": {
    "@monorep/utils": "workspace:*",
    "express": "^4.18.2"
  }
}
```

### 8.4 依赖管理最佳实践

1. **开发阶段**：使用 `workspace:*` 始终使用最新代码
2. **发布阶段**：pnpm 会自动将 `workspace:` 替换为实际版本号
3. **类型支持**：确保公共包导出类型定义文件

## 十、版本管理与发布

### 10.1 Changesets 简介

Changesets 是一个版本管理和发布工具，特点：
- 自动生成 CHANGELOG
- 语义化版本管理
- 支持 monorepo
- 自动更新依赖版本

### 10.2 安装配置

#### 安装

```shell
pnpm -Dw add @changesets/cli
pnpx changeset init
```

#### 配置文件 (.changeset/config.json)

```json
{
  "$schema": "https://unpkg.com/@changesets/config@3.0.0/schema.json",
  "changelog": "@changesets/cli/changelog",
  "commit": false,
  "fixed": [],
  "linked": [],
  "access": "restricted",
  "baseBranch": "main",
  "updateInternalDependencies": "patch",
  "ignore": []
}
```

### 10.3 使用流程

#### 1. 创建 changeset

```shell
pnpm changeset
```

交互式选择：
- 选择变更的包
- 选择版本类型（major/minor/patch）
- 填写变更说明

#### 2. 版本更新

```shell
pnpm changeset version
```

自动：
- 更新包版本号
- 生成 CHANGELOG.md
- 更新依赖版本

#### 3. 发布包

```shell
pnpm changeset publish
```

自动：
- 构建所有包
- 发布到 npm
- 创建 git tag

### 10.4 发布脚本

```json
{
  "scripts": {
    "changeset": "changeset",
    "version": "changeset version",
    "release": "pnpm build && changeset publish"
  }
}
```

### 10.5 版本管理策略

| 版本类型 | 说明 | 示例 |
|---------|------|------|
| **major** | 破坏性变更 | 1.0.0 → 2.0.0 |
| **minor** | 新功能（向后兼容） | 1.0.0 → 1.1.0 |
| **patch** | Bug 修复 | 1.0.0 → 1.0.1 |

## 十一、Docker 容器化

### 11.1 多阶段构建 Dockerfile

```dockerfile
# 基础依赖阶段
FROM node:18-alpine AS deps
WORKDIR /app
RUN corepack enable && corepack prepare pnpm@8.15.0 --activate
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY apps/backend/package.json ./apps/backend/
COPY apps/frontend/package.json ./apps/frontend/
COPY packages/*/package.json ./packages/
RUN pnpm install --frozen-lockfile

# 构建阶段
FROM node:18-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN pnpm build

# 后端生产阶段
FROM node:18-alpine AS backend
WORKDIR /app
COPY --from=builder /app/apps/backend/dist ./dist
COPY --from=builder /app/apps/backend/package.json ./
EXPOSE 3001
CMD ["node", "dist/index.js"]

# 前端生产阶段
FROM nginx:alpine AS frontend
COPY --from=builder /app/apps/frontend/dist /usr/share/nginx/html
COPY docker/nginx.conf /etc/nginx/nginx.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 11.2 Docker Compose 配置

```yaml
version: '3.8'

services:
  backend:
    build:
      context: .
      target: backend
    container_name: monorep-backend
    ports:
      - '3001:3001'
    environment:
      - NODE_ENV=production
    restart: unless-stopped

  frontend:
    build:
      context: .
      target: frontend
    container_name: monorep-frontend
    ports:
      - '80:80'
    depends_on:
      - backend
    restart: unless-stopped
```

### 11.3 Nginx 配置 (docker/nginx.conf)

```nginx
events {
  worker_connections 1024;
}

http {
  include /etc/nginx/mime.types;
  default_type application/octet-stream;

  server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    # 前端路由
    location / {
      try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
      proxy_pass http://backend:3001;
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
    }
  }
}
```

### 11.4 Docker 命令

```shell
# 构建并启动
```

### 12.2 发布工作流

#### .github/workflows/release.yml

```yaml
name: Release

on:
  push:
    branches: [main]

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - uses: pnpm/action-setup@v2
        with:
          version: 8.15.0
      
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
          registry-url: 'https://registry.npmjs.org'
      
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      
      - name: Build
        run: pnpm build
      
      - name: Create Release Pull Request or Publish

        with:
          publish: pnpm release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## 十三、最佳实践总结

### 13.1 项目结构原则

1. **按功能分层**：apps（应用）、packages（公共包）分离
2. **单一职责**：每个包只负责一个功能领域
3. **依赖方向**：apps 依赖 packages，packages 之间避免循环依赖

### 13.2 开发流程

```
1. 创建功能分支
   ↓
2. 开发功能（pnpm dev）
   ↓
3. 编写测试（pnpm test）
   ↓
4. 代码检查（pnpm lint）
   ↓
5. 类型检查（pnpm typecheck）
   ↓
6. 提交代码（pnpm commit）
   ↓
7. 创建 PR
   ↓
8. CI 自动检查
   ↓
9. 合并到主分支
   ↓
10. 创建 changeset
   ↓
11. 发布版本
```

### 13.3 性能优化建议

1. **使用 Turborepo 缓存**：加速重复构建
2. **并行执行任务**：充分利用多核 CPU
3. **增量构建**：只构建变更的包
4. **pnpm 硬链接**：节省磁盘空间

### 13.4 常见问题

#### 问题 1：依赖安装失败

```shell
# 清除缓存重新安装
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
```

#### 问题 2：类型检查错误

```shell
# 重新生成类型声明
pnpm build
pnpm typecheck
```

#### 问题 3：Turborepo 缓存问题

```shell
# 清除 Turborepo 缓存
pnpm turbo clean
rm -rf .turbo
```
